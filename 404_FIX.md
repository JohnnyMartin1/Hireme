# 🔧 404 Error Fix - Next.js Server Configuration

## Issue
The app is getting a 404 error when trying to load from `http://192.168.1.152:3000/`

## Root Cause
Next.js dev server might be binding to `localhost` only, which means it's not accessible via IP address.

## Solution

### Option 1: Start Dev Server with Hostname Binding (Recommended)

Update your `package.json` scripts to bind to all interfaces:

```json
"dev": "next dev -H 0.0.0.0"
```

This makes the server accessible via:
- `http://localhost:3000` (from your Mac)
- `http://192.168.1.152:3000` (from iOS simulator)

### Option 2: Use localhost with Port Forwarding

Keep using localhost but ensure the server is accessible.

## Quick Fix

1. **Stop your current dev server** (Ctrl+C)

2. **Start it with hostname binding:**
   ```bash
   next dev -H 0.0.0.0
   ```

3. **Or update package.json:**
   ```json
   "dev": "next dev -H 0.0.0.0"
   ```

4. **Then restart:**
   ```bash
   npm run dev
   ```

5. **Test the connection:**
   ```bash
   curl http://192.168.1.152:3000/
   ```
   Should return 200 OK, not 404.

6. **Rebuild the app in Xcode**

## Why This Happens

By default, Next.js binds to `127.0.0.1` (localhost only), which means:
- ✅ Accessible from `http://localhost:3000` on your Mac
- ❌ NOT accessible from `http://192.168.1.152:3000` (IP address)

Binding to `0.0.0.0` makes it accessible from:
- ✅ `http://localhost:3000`
- ✅ `http://127.0.0.1:3000`
- ✅ `http://192.168.1.152:3000`
- ✅ Any network interface
